## Common Security Questions

**Q: Why `-EncodedCommand` instead of temp files?**
A: Temp files can be swapped out before execution (TOCTOU race condition). `-EncodedCommand` passes the script directly to PowerShell as a process argument — atomic, no file race.

**Q: Why HMAC instead of just SHA-256?**
A: SHA-256 detects corruption but not intentional tampering (no key). HMAC uses a machine-specific key (hostname + username) so only the original machine can generate valid signatures.

**Q: Why crop screenshots before VLM?**
A: Reduces PII exposure. Full screen might contain passwords, file paths, or sensitive info. Bottom-right 70% × 60% region (from 40% down/30% right) is sufficient for accept buttons while hiding IDE title bar and left sidebar.

**Q: What if binary hash file is missing?**
A: Falls back to source files (`.mjs` versions) with a warning. Hash file should be generated by build script (`scripts/rebuild_core_bundle.mjs`).

**Q: How are profiles upgraded from old format?**
A: On load, legacy profiles (version 1 but no signature) are accepted. On next save, they're automatically signed with current HMAC key and written in envelope format.

## Logging Security Events

Watch logs for these indicators:

| Log Message | Severity | Action |
|-------------|----------|--------|
| `HMAC verification failed` | 🔴 CRITICAL | Profiles tampered — starting fresh |
| `hash mismatch` | 🟠 HIGH | Binary may be compromised — using source files |
| `Command is not allowed` | 🟠 HIGH | Attempted injection in runCommand — fix settings |
| `no safe label variants` | 🟡 MEDIUM | VLM suggested dangerous button — reject & investigate |
| `Image resized to X bytes` | 🟢 LOW | Normal: large screenshot was auto-reduced |
| `Cursor position X,Y` | 🟢 LOW | Normal: tracking user activity |

## Incident Response

**If you see "HMAC verification failed":**
1. Check file permissions on `firejumper_state/button_profiles.json`
2. Verify you haven't edited the file manually
3. If file was truly tampered: restore from backup
4. If legitimate: file was migrated from old format (normal on first run)

**If you see "hash mismatch":**
1. Check that `core-bin/core_bundle.jsc.sha256` matches your build
2. Regenerate hash: `node scripts/rebuild_core_bundle.mjs`
3. Never bypass hash checks with `-EncodedCommand` tricks

**If you see "Command is not allowed":**
1. Review your `vlmAutoClicker.runCommand` setting
2. Ensure it's a safe command: `node start.js`, `npm test`, etc.
3. Never add pipes, semicolons, or shell operators
